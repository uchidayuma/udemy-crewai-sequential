# UI/UX 設計仕様書: 架電レコメンドツール

## 1. 画面一覧と遷移図

```
[メイン画面: 架電優先リスト] --タップ--> [顧客詳細画面]
```

## 2. 各画面のコンポーネント構成

### 2.1 架電優先リスト画面（メイン画面）

```
+------------------------------------------+
|                   Header                 |
|------------------------------------------|
|   [検索バー]  [フィルターボタン]          |
|------------------------------------------|
| [顧客リスト（スクロール可能なリスト）]   |
|  +------------------------------------+  |
|  | # AIスコア | 顧客名 | 最終接触日    |  |
|  | 1         | 田中商事 | 2023/09/12   |  |
|  |------------------------------------|  |
|  | AIスコア   | 購入金額  | 商談確度    |  |
|  | 87         | ¥10,000,000 | 90%     |  |
|  +------------------------------------+  |
+------------------------------------------+
```

#### コンポーネント詳細

- **Header**: アプリのタイトルが表示されるナビゲーションバー。モバイルでの視認性を重視した大きさ。
- **検索バー**: 顧客名や情報での簡易検索が可能。
- **フィルターボタン**: フィルターオプションを呼び出すドロップダウン。
- **顧客リスト**: 優先順位付けされた顧客がカード形式で表示されるスクロール可能なリスト。顧客名カードをタップすると顧客詳細画面へ遷移。

### 2.2 顧客詳細画面

```
+------------------------------------------+
|                   Header                 |
|------------------------------------------|
| [顧客情報タブ選択]                        |
|  +------------------------------------+  |
|  | 顧客名: 田中商事                    |  |
|  | 住所: 東京都千代田区                |  |
|  | 電話番号: 03-xxxx-xxxx             |  |
|  +------------------------------------+  |
| [基幹データ | OCRデータ | その他情報]  |  |
|------------------------------------------|
| [トークスクリプトヒント]                  |
|  +------------------------------------+  |
|  | 新製品に興味がありそうなので次回  |  |
|  | の架電時に提案してみましょう。     |  |
|  +------------------------------------+  |
+------------------------------------------+
```

#### コンポーネント詳細

- **顧客情報タブ選択**: タブ形式で基幹システム、OCRデータ、その他の統合情報を切り替え表示可能。
- **トークスクリプトヒント**: AIが生成した対話の鍵を示すセクション。モバイルでの閲覧を考慮し、スクロールなしで表示可能。

### 2.3 検索・フィルター機能

```
+------------------------------------------+
|                   Header                 |
|------------------------------------------|
|   [担当支店]  [重点フラグ]  [確度]       |
|   [Clear Filters Button]                 |
+------------------------------------------+
```

- **フィルター項目**: チェックリスト、およびスライダーを用いて柔軟な絞り込み操作を提供。
- **Clear Filters Button**: フィルター設定をワンタップで初期化。

## 3. モバイルファースト設計の根拠と考慮事項

- **単一画面に重要情報を集約**: モバイル環境での迅速な情報取得を可能にするレイアウト。
- **視認性の高いフォントとボタンサイズ**: 検索バーやフィルターなど、頻繁に利用するUI要素を操作しやすい大きさに設定。
- **レスポンシブデザイン最適化**: 各種スマートフォンの画面サイズでの操作性を確認。

## 4. IT リテラシーが高くない営業担当者向けの UX 配慮事項

- **直感的なナビゲーション**: 単純で統一されたタップ操作。
- **視覚的階層化**: 情報の整理と優先順位付けを直感的に理解できる配色とレイアウト。
- **ヘルプアイコン**: 各画面の主要操作で、タップすると簡易説明を表示する。

## 5. トークスクリプトヒントの表示 UI コンセプト

- **ナチュラルランゲージでの指示**: AIが生成するヒントを「次の行動に役立つ」形式で提示。
- **強調表示**: トークスクリプトの重要部分を太字にするなど視覚的に強調。
- **背景色の調整**: メイン画面のトークスクリプトヒント部分はソフトな背景色で他の情報と区別。

この設計仕様書は、エンドユーザーである営業担当者の使いやすさを最優先に考慮し、モバイル環境での実用性を高めることを目的としています。
