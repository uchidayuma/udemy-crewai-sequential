# プロジェクト仕様：架電レコメンドツール開発

## 発注元企業
- 会社名: 株式会社ルート・テック (Route Tech Inc.)
- 立ち位置: 東証プライム上場・総合製造メーカー「ルート・ホールディングス」のIT戦略子会社（SIer）
- 従業員数: 850名（うちエンジニア600名）
- 組織文化: ウォーターフォール型開発が主体。上長の承認が絶対。プロトタイプにも最低限の設計書とリスク説明が必要。

## 親会社からの正式依頼メール（要約）
依頼者: 親会社 営業推進部 佐藤氏
期限: 10日後の役員報告（予算承認に直結）

課題:
1. 営業リストの精度不足: 約5,000社の顧客リストがあるが、架電先の選定は各担当者の勘に依存。成約率は2%以下。
2. データの分散: 購入履歴（基幹システム）・商談メモ（支店Excel）・展示会リード（名刺OCR）がバラバラに存在。
3. 判断の遅れ: どの会社に・いつ・どの製品を提案すべきか、AIで自動化したい。

依頼内容:
- 散らばったデータを統合するデータベース設計案
- AIが次に架電すべき顧客を優先順位付けするロジックの提示
- 営業担当者が一目で理解できる優先リスト管理画面のイメージ

## 現場ヒアリング結果（営業担当者3名）
- 課題A（優先順位）: 毎朝どの顧客に電話すべきか決めるのに30分以上かかる。前回連絡から時間が空いていて購入金額が大きい顧客を自動で教えてほしい。
- 課題B（情報検索）: 外出先でスマホから、その顧客の購買履歴と最後の商談内容を即座に確認したい。基幹システムは重くて使えない。
- 課題C（きっかけ）: 「電話しろ」だけでは困る。AIが「部品の交換時期です」「新製品がこの顧客のニーズに合っています」といったトークスクリプトのヒントを提示してほしい。
