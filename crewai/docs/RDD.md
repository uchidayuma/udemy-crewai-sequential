```markdown
# 製品要件定義書 (RDD): 架電レコメンドツール開発

## 1. プロジェクト背景・目的
本プロジェクトの目的は、分散している顧客データを統合し、AIを用いて架電優先順位付けを自動化することで、営業活動の効率を向上することである。主な依頼内容は、データベース設計、架電優先順位付けロジック、営業担当者が容易に利用可能なリスト管理画面の提供である。

## 2. ステークホルダー一覧
- **依頼元**: 親会社 営業推進部 佐藤氏
- **利用者**:
  - 営業担当者
  - 各支店の営業マネージャー
- **承認者**:
  - プロジェクトマネージャー（Orchestrator）
  - 部長クラスの上長

## 3. ユーザーストーリー
- **課題A（優先順位）**: 営業担当者として、毎朝どの顧客に電話をすべきか、前回連絡から時間が空き、購入金額が大きい顧客を自動で特定してほしい。
- **課題B（情報検索）**: 営業担当者として、外出先でスマホからその顧客の購買履歴と最後の商談内容を即座に確認したい。
- **課題C（きっかけ）**: 営業担当者として、AIが提供する「部品の交換時期」「新製品提案のヒント」などのトークスクリプトを得たい。

## 4. 機能要件リスト
- **Must Have**:
  - データソースの統合とデータベース設計
  - AIによる架電優先順位付けロジック
  - 営業担当者向けの優先リスト管理画面
  - 重要顧客へのトークスクリプト自動生成機能
- **Should Have**:
  - 外出先でのスマートフォン対応
- **Could Have**:
  - マイクロサービスアーキテクチャへの対応
- **Won't Have**: 
  - 大幅なシステムインフラの刷新

## 5. 非機能要件
- **レスポンス速度**: システムの応答時間は2秒以内
- **可用性**: 95%以上の稼働率を確保
- **モバイル対応**: スマートフォンからのアクセスに最適化
- **セキュリティ**: 業界標準の認証と権限管理を採用

## 6. データ品質リスク
- **NULL値**: データソースAの最終購入日にNULL
- **表記揺れ**: データソースBの企業名全角/半角揺れ
- **未登録データ**: データソースCの未構造化テキスト

## 7. 用語集
- **確度**: 商談の進行具合と成約期待値
- **重点フラグ**: ビジネス優先度を示すための指標
- **リード**: 潜在的な新規顧客の情報

## 8. スコープ外の明示
- 今回のプロトタイプでは、全社の基幹システム改修には対応しない。
- 長期的な運用方針やシステム維持管理に関する詳細提案は含まない。

```

この製品要件定義書は、架電レコメンドツールを開発する上で必要な要件を明確に記載しています。特にステークホルダーのニーズやデータの品質リスクを細かくアセスメントし、具体的な解決策を提示しています。部長クラスの判断を得る際に備え、誠実で確かなトーンを保ちつつ、開発チームが迷わず実装に着手できる内容としています。
